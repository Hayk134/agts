# Внимание!

Предполагается, что все работы в рамках использования окружения производятся в ОС Windows 11 24H2 или новее.

## Введение

В данном репозитории содержится исходный код и инструкции, необходимые для разворачивания и использования модуля
Клиента для АНТС.

Клиент необходим для взаимодействия с Автопилотом и Доверенным модулем АНТС, а именно:

- ввода маршрутного задания
- обработки задач робота в процессе выполнения заезда
- создания и запуска обработчиков кода, нейтрализующего воздействие киберпрепятствий.

## Работа с системой

В репозитории предоставляется уже готовое, собранное решение для запуска всего окружения.
Для этого вам всего потребуется:

```shell
# Инициализация системы (при первом запуске) в директории с клиентом АНТС 
uv sync
```

### 1. Запустить сервер Клиента АНТС

```shell
# Команды выполняются в директории с клиентом АНТС 
uv run jupyter notebook
```

### 2. Запустить сервер ЦД

```shell
# Команды выполняются в директории с ЦД АНТС 
./twin.exe
```

### 3. Запустить сервер АСО АНТС

```shell
# Команды выполняются в директории с АСО АНТС 
uv run launcher.py --twin
```

## Решение задачи

### 1. Изучите дополнительную документацию

Изучите все файлы в директории ```docs``` - в них вы найдёте описание самого полигона, происходящих там событий и
карту полигона.

> Тщательно изучите эти документы - это потребуется вам при выполнении основной соревновательной задачи!

### 3. Приступите к решению задачи

И вот, практическая часть вашей работы - написание кода для решения задач робота и кибериммунности!

Решение задач подразумевает редактирование файлов в директории `src`, в субдиректории с названием UserCode.

# Важно - конкурсная информация

Напоминаем, что вам запрещено редактировать и/или создавать любые файлы кроме расположенных в директориях модулей, и
имеющих названия, соответствующие шаблону `mod_X_code.ipynb` и `mod_X_code.py`, где `X` - литера ТЕКУЩЕГО модуля (
транслитом).

Также, вам запрещено читать (в рамках работы программы, не относится к текстам исходного кода), перезаписывать, удалять
или любым образом модифицировать порядок работы любых существующих (не написанными вами и имеющихся в репозитории
изначально) объектов кода кроме тех, доступ к которым разрешён в явном виде (в ноутбуках для модулей). В случае
нарушения этого правила, заезд будет признан неуспешным. Исключением являются случаи, согласованные ГЭ в процессе
выполнения модулей.

Также напоминаем, что вы можете изменять код решений для задач робота с целью решения задач кибериммунности.
